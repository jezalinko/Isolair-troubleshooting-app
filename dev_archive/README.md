# Development Archive

This folder contains **legacy and one-off development artifacts** that were used during the early stages of building the Isolair Troubleshooting App.

Nothing in this directory is required for normal operation of the application.

---

## Purpose

The `dev_archive/` exists to:

- Preserve historical context and development work
- Keep the main repository clean and intentional
- Avoid deleting potentially useful reference material
- Clearly separate **active tooling** from **legacy scripts**

If you are working on or running the application, you generally do **not** need anything in this folder.

---

## Contents

Typical contents may include:

- **Legacy extraction scripts**
  - One-time scripts used to extract or convert data from PDFs
  - Early automation or test utilities
- **Prototype or deprecated logic**
  - Scripts or YAML files that are no longer part of the active flow
- **Experimental or exploratory work**
  - Code kept for reference but not actively maintained

Each subfolder should be self-describing where possible.

---

## Active Code Location

If you are looking for **current, supported tools**, see instead:

- `scripts/` — active development and validation tools
- `data/yaml/` — authoritative troubleshooting logic
- `docs/` — current documentation and reference material

---

## Maintenance Notes

- Files in this directory are **not guaranteed to be up to date**
- No compatibility with the current schema or app runtime is implied
- Items may be removed or reorganised as the project evolves

This folder exists for **historical and reference purposes only**.
